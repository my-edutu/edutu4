# Firebase Authentication Setup Guide\n\n## Prerequisites\n\n1. Create a Firebase project at [Firebase Console](https://console.firebase.google.com/)\n2. Enable Authentication and Firestore in your Firebase project\n\n## Firebase Console Setup\n\n### 1. Enable Authentication Methods\n\n1. Go to **Authentication** > **Sign-in method**\n2. Enable **Email/Password** authentication\n3. Enable **Google** authentication:\n   - Click on Google provider\n   - Enable it\n   - Add your project's authorized domains\n\n### 2. Configure Firestore Database\n\n1. Go to **Firestore Database**\n2. Click **Create database**\n3. Choose **Start in test mode** (for development)\n4. Select a location for your database\n\n### 3. Set up Firestore Security Rules\n\nReplace the default rules with:\n\n```javascript\nrules_version = '2';\nservice cloud.firestore {\n  match /databases/{database}/documents {\n    // Users can only read/write their own user document\n    match /users/{userId} {\n      allow read, write: if request.auth != null && request.auth.uid == userId;\n    }\n  }\n}\n```\n\n## Environment Variables\n\nYour `.env` file is already configured with your Firebase credentials:\n\n```env\nVITE_FIREBASE_API_KEY=AIzaSyBfUyZI0N58KAusra7eLz2Cjfq22zcAFw0\nVITE_FIREBASE_AUTH_DOMAIN=edutu-3.firebaseapp.com\nVITE_FIREBASE_PROJECT_ID=edutu-3\nVITE_FIREBASE_STORAGE_BUCKET=edutu-3.firebasestorage.app\nVITE_FIREBASE_MESSAGING_SENDER_ID=36956735687\nVITE_FIREBASE_APP_ID=1:36956735687:web:8a4b0c2b4e52c647cb16e2\nVITE_FIREBASE_MEASUREMENT_ID=G-YGNN6SQJRE\n```\n\n## Features Implemented\n\n✅ **Email/Password Authentication**\n- Sign up with email, password, name, and age\n- Sign in with email and password\n- Form validation (email format, password length, age range 16-30)\n\n✅ **Google OAuth Authentication**\n- One-click sign in with Google\n- Automatic user document creation\n\n✅ **User Data Persistence**\n- Automatic user document creation in Firestore\n- Fields: `name`, `email`, `age`, `createdAt`, `uid`\n\n✅ **Security & Validation**\n- Client-side form validation\n- Server-side validation via Firebase Auth\n- No hardcoded credentials\n- Environment variables for sensitive data\n\n✅ **UX Improvements**\n- Loading states during authentication\n- Disabled buttons while processing\n- Friendly error messages\n- \"Remember me\" checkbox\n- Auto-login for returning users\n\n✅ **Future-Ready Architecture**\n- Modular auth service (`src/services/authService.ts`)\n- Validation utilities (`src/utils/validation.ts`)\n- Authentication hook (`src/hooks/useAuth.ts`)\n- Easy to extend for phone auth, password reset, etc.\n\n## File Structure\n\n```\nsrc/\n├── config/\n│   └── firebase.ts          # Firebase configuration\n├── services/\n│   └── authService.ts       # Authentication service\n├── utils/\n│   └── validation.ts        # Form validation utilities\n├── hooks/\n│   └── useAuth.ts          # Authentication state hook\n└── components/\n    └── AuthScreen.tsx       # Updated authentication UI\n```\n\n## Usage\n\n### In your main App component:\n\n```typescript\nimport { useAuth } from './hooks/useAuth';\n\nfunction App() {\n  const { user, userData, loading, isAuthenticated } = useAuth();\n\n  if (loading) {\n    return <div>Loading...</div>;\n  }\n\n  if (!isAuthenticated) {\n    return <AuthScreen onGetStarted={(userData) => {\n      // User is now authenticated\n      console.log('User authenticated:', userData);\n    }} />;\n  }\n\n  return (\n    <div>\n      <h1>Welcome, {userData?.name}!</h1>\n      {/* Your app content */}\n    </div>\n  );\n}\n```\n\n### To add logout functionality:\n\n```typescript\nimport { authService } from './services/authService';\n\nconst handleLogout = async () => {\n  try {\n    await authService.signOut();\n  } catch (error) {\n    console.error('Error signing out:', error);\n  }\n};\n```\n\n## Adding Future Features\n\n### Phone Authentication\n\n1. Enable Phone provider in Firebase Console\n2. Add phone auth methods to `authService.ts`\n3. Update validation utilities\n4. Add phone input to AuthScreen\n\n### Password Reset\n\n1. Add `sendPasswordResetEmail` method to `authService.ts`\n2. Create password reset UI component\n3. Add \"Forgot Password\" flow\n\n### Email Verification\n\n1. Add `sendEmailVerification` method to `authService.ts`\n2. Check `user.emailVerified` status\n3. Add verification reminder UI\n\n## Testing\n\n1. Start the development server: `npm run dev`\n2. Try signing up with a new email\n3. Try signing in with existing credentials\n4. Test Google OAuth\n5. Check Firestore console to see user documents\n6. Test form validation with invalid inputs\n\n## Security Notes\n\n- Never commit `.env` files to version control\n- Set up proper Firestore security rules for production\n- Consider implementing rate limiting for authentication\n- Use Firebase App Check for additional security in production
